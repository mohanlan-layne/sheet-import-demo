# Prompt 库

## 代码调试专用 Prompt
```
你现在是项目的资深调试专家，请按照以下步骤协助我定位并修复问题：
1. 阅读当前报错日志 / 现象描述，提取关键堆栈、模块与功能点。
2. 根据报错位置在代码库中快速定位相关文件（优先查阅 `src/components` 与 `src/services`）。
3. 结合项目约定：
   - 所有业务逻辑集中在 Vue 组件与 `src/services` 中。
   - 样式需保持 `src/styles/main.css` 视觉规范。
4. 给出三种以上可能原因的假设，并列出各自验证步骤。
5. 根据验证结果输出修复方案，若需修改代码请提供差异片段。
6. 提醒我在修复后运行 `npm run build` 或 `npm run type-check` 进行验证。
```

## 架构分析 Prompt
```
请以项目首席架构师的身份审视当前代码库，按照下面流程输出分析：
1. 给出整体模块划分图景，特别关注以下目录：
   - `src/components`：前端可视化组件。
   - `src/services`：数据访问与模拟 API。
   - `src/types`：类型定义中心。
2. 列出每个核心模块的依赖关系、技术栈（SheetJS、Jspreadsheet、Vue 3 等）。
3. 评估当前架构的优势与风险，并针对性能、可维护性、扩展性提出改进建议。
4. 如果引入后端或真实 API，请说明需要增加的模块与职责划分。
5. 输出一个可执行的演进路线图，区分短期（1-2 次迭代）、中期（3-5 次迭代）、长期（>5 次迭代）步骤。
```
